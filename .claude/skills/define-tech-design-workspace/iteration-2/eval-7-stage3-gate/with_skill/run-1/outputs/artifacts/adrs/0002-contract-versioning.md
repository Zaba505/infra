## Context and Problem Statement
How do we version the platform contract to support TR-02 (multiple concurrent versions)?

## Considered Options
* Semver in contract package path
* Date-based versioning
* Single rolling version with feature flags

## Decision Outcome
Proposed: semver in contract package path. Awaiting confirmation.
